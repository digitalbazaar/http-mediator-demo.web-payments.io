Payment App Demo (HTTP API)

This site demonstrates a basic implementation of a payment application , which can be used to receive a payment request and produce a payment response. This demo is designed for non-interactive payments use cases. For example, auto payment of utility bills, business-to-business payments, a car paying for a parking spot (Internet of Things), etc. For more information, see the Web Payments HTTP API specification.

If you just want to try the Web Payments HTTP API out, the Payment Mediator Demo is probably the best place to start.

If you are a software developer and need a payment application that follows the HTTP API, the following things may be of interest to you:

  • Setup a new payment app. Try curl {{$root.app.config.data.baseUri}}/apps/legacy-card
  • Submit a payment request. Do an HTTP POST of a payment request to {{$root.app.config.data.baseUri}}/apps/legacy-card/{your-payment-app-id}.

The source for this demonstration can be found on Github under the digitalbazaar/http-mediator-demo.web-payments.io project.
